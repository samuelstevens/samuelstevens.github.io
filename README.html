Build/Serve with Go

The fish scripts were replaced by a small Go CLI that uses github.com/bitfield/script to orchestrate shell commands.

Prereqs - Go 1.23+ - pandoc - uv (optional; used for the dev server; falls back to python3/python)

Default output directory - $HOME/Development/samuelstevens.github.io (override with -output PATH)

Common commands

# Build everything into the output directory
go run ./cmd/build build-all

# Serve the built site at http://localhost:8042
go run ./cmd/build serve

# Watch the repo and rebuild the newest Markdown page on change
go run ./cmd/build watch            # use -interval 300ms to change poll rate

# Publish: clean output, build-all, then git add/commit/push in the output repo
go run ./cmd/build -v publish

# Build just one file
go run ./cmd/build htmlify microblog/index.md

# Use a different output location
go run ./cmd/build -output /tmp/site build-all
go run ./cmd/build -output /tmp/site serve

Tip: run serve and watch in separate terminals.

Notes - publish requires the output directory to be a git repo with a configured remote. - The server prefers uv run python -m http.server; if uv is not found it falls back to python3 or python.

Elm

  1. python -m src.scripts.conferences src/data/conferences.json
  2. Copy the output to src/Conferences.elm
  3. elm make src/Conferences.elm --output js/conferences.js
